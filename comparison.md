# Stack comparison

Every backend implements the same contract ([`spec/openapi.yaml`](spec/openapi.yaml)) and is verified identically by [`conformance/`](conformance/). Internals stay idiomatic per stack — so the tables below show where the stacks **diverge** while producing the **same wire behavior**.

## Stack & tooling

| | go-net-http | kotlin-spring | python-django | python-fastapi | ts-nestjs |
|---|---|---|---|---|---|
| Language / framework | Go 1.26 · stdlib `net/http` | Kotlin 2.2 · Spring Boot 4 + MVC | Python 3.14 · Django 6 + DRF | Python 3.14 · FastAPI (async) | TypeScript · NestJS 11 |
| Dependency manager | Go modules (`go.sum`) | Gradle (Kotlin DSL) | uv (`uv.lock`) | uv (`uv.lock`) | npm (`package-lock.json`) |
| Data layer | sqlc + pgx (hand-written SQL, no ORM) | Spring Data JPA + Hibernate 7 | Django ORM + DRF | SQLAlchemy 2.0 (async) + asyncpg | TypeORM |
| Dev server / reload | `go run .` (Compose Watch `sync+restart`) | `bootRun` (Compose Watch `sync+restart`) | `runserver` (Compose Watch) | uvicorn `--reload` (Compose Watch) | `nest start --watch` (Compose Watch) |
| Migrations | hand-written SQL, goose (`go:embed`) | hand-written SQL, Flyway (forward-only) | `makemigrations`, auto-named | Alembic, autogenerated | TypeORM CLI, manually named |

## Data model (`User`)

| | go-net-http | kotlin-spring | python-django | python-fastapi | ts-nestjs |
|---|---|---|---|---|---|
| UUIDv7 id (DB-side `uuidv7()`) | hand-written DDL | Flyway DDL + `@Generated` read-back | `db_default=Func('uuidv7')` | `server_default=text('uuidv7()')` | `@PrimaryColumn(default: () => 'uuidv7()')` |
| Timestamps (DB default `now()`) | hand-written DDL | Flyway DDL + `@Generated` read-back | `db_default=Func('now')` | `server_default=func.now()` | `@CreateDateColumn` / `@UpdateDateColumn` |
| `timestamptz` | explicit (migration DDL) | explicit (Flyway DDL) | free (`USE_TZ=True`) | explicit (`DateTime(timezone=True)`) | explicit (`type: 'timestamptz'`) |
| snake_case columns | hand-written in SQL | free (Spring naming strategy) | free (field name *is* the column) | free (mapped attr name *is* the column) | custom `SnakeNamingStrategy` |
| Table name (`users`) | hand-written DDL | `@Table(name = "users")` | `Meta.db_table` | `__tablename__` | `@Entity('users')` |
| Create round-trips | 1 — `INSERT … RETURNING` | 1 — `INSERT … RETURNING` | 1 — `INSERT … RETURNING` | 1 — `INSERT … RETURNING` | 1 — `INSERT … RETURNING` |

## Contract behavior (same result, different mechanism)

| | go-net-http | kotlin-spring | python-django | python-fastapi | ts-nestjs |
|---|---|---|---|---|---|
| Input validation | stdlib `net/mail.ParseAddress` | Bean Validation (`@Email`) | DRF serializer (`EmailField`) | `email-validator` (preserves original) | class-validator + `ValidationPipe` |
| Duplicate email → `409` | `errors.As` → `*pgconn.PgError`, `Code == "23505"` | catch `DataIntegrityViolationException`, SQLState `23505` | catch `IntegrityError`, `__cause__.sqlstate == '23505'` | catch `IntegrityError`, `orig.sqlstate == '23505'` | catch `QueryFailedError`, `driverError.code === '23505'` |
| Malformed id → `404` | `uuid.Parse` error → `404` | `UUID.fromString` → `IllegalArgumentException` → `404` | DRF `get_object_or_404` (rejects bad UUID) | `UUID(str)` → `ValueError` → `404` | `ParseUUIDPipe({ errorHttpStatusCode: NOT_FOUND })` |
| RFC 9457 errors | `writeProblem` helper | native `ProblemDetail` + `@ControllerAdvice` | DRF `EXCEPTION_HANDLER` + JSON-only renderer | exception handlers (`problem.py`) | global exception filter (`@Catch()`) |
| Host validation | none (stdlib) | none (servlet) | `ALLOWED_HOSTS` (env-driven) | none (ASGI) | none (Express) |
| Routed endpoints | `createUser` + `getUser` (explicit `mux` registration) | `create` + `get` (CRUD via `JpaRepository`, unrouted) | `create` + `retrieve` (mixins on `GenericViewSet`) | `create_user` + `get_user` (explicit `@app` routes) | `create` + `findOne` (rest in service, unrouted) |

All five reach the duplicate-detection result through the **same Postgres signal** (SQLSTATE `23505`) via different language APIs — a good example of "identical contract, idiomatic internals."

A FastAPI-specific catch: Pydantic's `EmailStr` *normalizes* addresses (punycode → Unicode), which made the response email violate the spec's `format: email` — surfaced by conformance and fixed by validating without rewriting the input, matching how the other stacks behave on the wire.

All five create a user in a **single `INSERT … RETURNING` round-trip** (verified by Postgres `log_statement`): every stack now uses DB-side defaults and fetches the server-generated id/timestamps in the same statement. (python-django originally generated those values app-side and used a plain `INSERT`; it was moved to DB-side defaults via `db_default` to match `spec/schema.sql`. TypeORM's `save()` uses `RETURNING` here rather than the insert-then-`SELECT` it's sometimes assumed to do. kotlin-spring's Hibernate emits `INSERT … RETURNING` via `@Generated` on the DB-defaulted columns, with no follow-up `SELECT`.)

`spec/schema.sql` is the persistence source of truth; bringing python-django into line with it required modest, supported overrides against Django's grain — `db_default=Func('now')` (its idiomatic `Now()` renders `statement_timestamp()`, and `auto_now_add`/`auto_now` are app-side), `db_default=Func('uuidv7')` (no built-in UUIDv7 expression), `Meta.db_table = 'users'`, and `TextField` over `EmailField` (email validation stays in the serializer's explicit `EmailField`). All achievable, with the timestamp default the only real friction point.

kotlin-spring (Spring Boot 4) surfaced two stack-specific notes. Boot 4 split autoconfiguration into per-technology modules, so Flyway needs `spring-boot-flyway` alongside `flyway-core` — having Flyway on the classpath no longer triggers it. And Flyway's versioned migrations are forward-only (no down/rollback in Community), the only stack here without reversible migrations. Its error layer is the cleanest of the five — Spring's `ProblemDetail` is RFC 9457-native — though it adds an `instance` field the others omit (harmless; the contract's `Problem` schema allows extra properties).
